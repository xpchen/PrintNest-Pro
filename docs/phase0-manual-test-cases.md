## 0-A 大文件保护 + sharp 缩略图

### TC-A1: 正常图片导入生成缩略图
1. 打开项目，进入「资源」面板
2. 导入一张 5MB 左右的 PNG/JPG 图片
3. **预期：**
   - 导入成功，图片出现在资源面板
   - 项目 `assets/` 目录下生成 `xxx_thumb.png` 缩略图文件
   - 缩略图最长边 ≤ 800px
   - 数据库 `assets` 表中 `pixel_width` 和 `pixel_height` 有值

### TC-A2: 超大文件拦截
1. 准备一张 > 50MB 的文件（可用任意大文件改扩展名）
2. 尝试导入
3. **预期：**
   - 导入流程中进行文件大小检查
   - 控制台 warn 日志记录文件超限

### TC-A3: readAsBase64 大文件保护
1. 已导入一张图片，但手动将其替换为 >50MB 文件（保持路径不变）
2. 触发 readAsBase64
3. **预期：** 返回空或报错，不会占用大量内存

## 0-H 模板域 Undo/Redo

### TC-H1: 模板编辑撤销
1. 进入模板模式
2. 添加一个文本元素
3. 修改其文本内容为 "A"
4. 再修改为 "B"
5. 按 Ctrl+Z（Mac: Cmd+Z）
6. **预期：** 文本回退到 "A"

### TC-H2: 重做
1. 上一步基础上，按 Ctrl+Shift+Z（Mac: Cmd+Shift+Z）
2. **预期：** 文本恢复为 "B"

### TC-H3: 模板与排版 undo 互不干扰
1. 在模板模式下做几步编辑
2. 切换到排版模式，做一些排版操作
3. 按 Ctrl+Z
4. **预期：** 只撤销排版操作，不影响模板状态
5. 切回模板模式，按 Ctrl+Z
6. **预期：** 只撤销模板编辑，不影响排版

### TC-H4: 输入框中不拦截
1. 在 Inspector 的文本输入框中聚焦
2. 按 Ctrl+Z
3. **预期：** 浏览器原生撤销（只撤销输入框内容），不触发模板历史撤销

### TC-H5: 新操作清空 redo 栈
1. 做操作 A → B → C，撤销两步到 A
2. 做新操作 D
3. 按 Ctrl+Shift+Z
4. **预期：** 无可重做的操作（B、C 已从 redo 栈清除）

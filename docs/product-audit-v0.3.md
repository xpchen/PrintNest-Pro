## 二、功能性缺陷（影响核心流程）

### 1. 模板画布交互不完整
- 元素只能在 SVG 内拖动，无法调整大小（缺 resize handle）
- 无对齐辅助线
- 无复制/粘贴元素
- 无 Ctrl+Z 撤销（模板编辑不在 Zundo 范围内）
- 文字元素无法实际输入文字内容（只有绑定字段设置）
- 图片元素无法选择/预览实际图片

### 2. 模板 → 排版链路薄弱
- 实例化后无进度反馈
- 实例卡片缺少预览缩略图
- 送入排版后，模板实例在画布上只显示色块（无图文渲染）
- 无法从排版结果反向定位到模板/数据记录

### 3. 输出中心骨架化
- 导出预设编辑无表单 UI
- 画布缩略图不显示实际排版内容
- 命名规则未实现
- 导出历史无文件大小/状态显示

### 4. 数据完整性风险
- 资产文件无去重（hash 列存在但从未写入）
- readAsBase64 无文件大小限制（大图可耗尽内存）
- 多窗口/并发保存无冲突检测

---

## 三、体验性缺陷（影响专业感知）

### 1. 缺少的基础交互
- 无全局键盘快捷键（Ctrl+S/Ctrl+Z/Delete/空格抓手）
- 无右键上下文菜单
- 无拖放导入（拖图片到窗口直接导入）
- 无确认对话框（删除等危险操作无提示）

### 2. 视觉打磨
- 画布缩放无平滑动画
- 模板画布 SVG 元素无阴影/边框区分
- 加载状态缺失
- Toast 太简陋

### 3. 数据展示
- 排版画布的图片只用 base64 data URL
- 资源列表无缩略图
- 无排版统计仪表盘

## 七、核心结论

后端/引擎成熟度已明显高于前端可交付完成度。真正的短板是模板编辑器、模板到排版的可视化链路，以及输出中心。这不是"还缺一些功能"，而是"产品最容易被用户感知的那层，完成度落后于底层能力"。

**v0.4 方向：** 建立 WYSIWYG 信任链，让用户能在模板编辑器里做出真实内容，在排版画布里看到可识别的结果，在输出中心完成闭环。
